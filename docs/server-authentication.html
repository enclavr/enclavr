Server Authentication

JWT and OIDC authentication implementation.

JWT Authentication

The server uses JWT tokens for API authentication.

Token Types

  • Access Token - Short-lived (24h), used for API requests
  • Refresh Token - Long-lived (7 days), used to get new access tokens

Middleware

JWTAuth middleware validates tokens on protected routes.

router.Use(middleware.JWTAuth())

OIDC Authentication

Server supports OIDC providers for SSO.

Configuration

OIDC_CLIENT_ID=...
OIDC_CLIENT_SECRET=...
OIDC_ISSUER=...
OIDC_REDIRECT_URL=...

Flow

  1. User visits /api/auth/oidc/login
  2. Redirected to OIDC provider
  3. Provider redirects to /api/auth/oidc/callback
  4. Server creates/updates user and returns JWT
